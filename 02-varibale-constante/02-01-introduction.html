JS - II 01 Introduction

Qu'est ce qu'une variable ?

Une variable est un conteneur servant à stocker des informations de manière temporaire, du texte une (chaine de caractère) ou un nombre. La variable varie ! elle stocke différentes valeurs au fil du temps

Une variable avec la valeur sont deux choses différentes

2- Règles de déclaration

On dit (comme en php) que l'on "déclare" une varibale. On déclaré une varibale avec les mots cléfs :

  • var
  • code
  • Le nom d'une variable doit obligatoirement commencer par une lettre ou un underscore _
  • Le nom d'une variable ne doit pas contenir que des lettres, des chiffres et des _
  • Il n'y a pas d'espace
  • voir introduction.js

3- Initialiser une variable

Quand on assigne une valeur à une variable vide on dit que l'on stocke valeur ou qu'on initialise une variable

4- Différences entre "var" et "let"

let est plus récent que var

let a une porté locale, alors que vara une portée globale

voir cette Vidéo

  • A - La remontée ou hoisting des variables

    Avec var on n'est pas obligé de déclarer la variable avant de la manipuler ou de l'affecter

  • B- La redéclaration des variables

    Avec var on avait le droit de déclarer plusieurs fois la même variable en utilisant à chaque fois var (ce qui avait pour effet de modifier la valeur stockée)

  • C- LA "portée" des variables

    La "portée" d'une variable désigne l'endroit ou cette variable va pouvoir être utilisée dans un script.

    Nous reparlerons donc de cette portée des variables plutard

  • Conclusion- Let ou Var ?

    La syntaxe de déclaration des variables avec let correspond à la nouvelle syntaxe. La syntaxe var est l'ancienne syntaxe qui est vouée à disparaitre

    Vous devriez donc aujourd'hui toujours utiliser e mot clef let pour déclarer vos variables

5- Quelle utilité pour les variables ?

Il sera pratique de stcoker différents types de données ou d'informations dans une variable

Exemple avec la fonction prompt on récupérera une donnée envoyé par un utilisateur, on fera un script qui contiendra une variable qui sera stocké quand elle sera envoyé
