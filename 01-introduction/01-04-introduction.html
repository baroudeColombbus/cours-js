L'ordre d'éxécution du code par le navigateur

Par défault, un navigateur va lire et éxécuter le code dans l'ordre de son écriture.

Lorsque le navigateur arrive soit chargé à un script, il va stopper le traitement du reste du html Jusqu'à ce que le code Javascript soit chargé dans la page et éxécuter

  1. Le navigateur commence à analyser ou à traîter le code HTML
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et téléchargé (si il est en externe) puis exécuté. Durant tout ce temps l'analyseur du navigateur bloque l'affichage du HTML ce qui peut ralentir, si le script est long,l'affichage de la page
  4. Dès que JS a été exécuté, le contenu HTML est affiché

Ce souci d'attente est résolu grâce au téléchargement aschynchrone des données avec l'attribut async ou defer dans le cas ou l'on fait appel à des fichiers externes.

Le point virgule

Le point virgule ; est utilisé pour signifier la fin d'une instruction (une variable, une condition, une boucle etc..)
